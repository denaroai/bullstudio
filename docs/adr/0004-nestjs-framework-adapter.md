# NestJS framework adapter

Bullstudio will add `@bullstudio/nestjs` as a framework adapter package for embedded mode. The package will expose a Nest-native `BullstudioModule.forRoot()` and `BullstudioModule.forRootAsync()` API, require an explicit `mountPath`, accept Bullstudio queue adapters rather than Nest-specific queue names, and support both Nest's Express and Fastify platforms by delegating to the existing Express and Fastify framework adapters.

This keeps NestJS in the same framework adapter family as Hono, Express, Fastify, and Next.js while avoiding separate Nest platform packages or duplicated embedded-core request handling. The NestJS package will statically depend on the Bullstudio Express and Fastify adapters, while Express and Fastify themselves will be optional peer dependencies so host applications only need the platform they already use. The first version will support one `BullstudioModule` mount per NestJS application so duplicate dynamic module imports fail clearly instead of creating ambiguous route registration and lifecycle behavior, and `forRootAsync()` will support `useFactory` with optional `inject` rather than the full Nest async provider shape.
